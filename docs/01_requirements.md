# VDOT Calculator - 要件定義書 (PRD)

## 1. 製品概要

### 1.1 製品名
**VDOT Calculator** - マラソンタイム予測アプリ

### 1.2 製品ビジョン
ランナーが自身の走力を把握し、目標レースに向けた適切なペース設定ができるツールを提供する。

### 1.3 対象ユーザー
- マラソン初心者〜中級者ランナー
- 練習ペースの指標を求めるランナー
- レースタイムの予測をしたいランナー

---

## 2. ビジネス要件

### 2.1 背景
マラソンのタイム予測には様々な手法があるが、Jack Daniels 氏が開発した VDOT 式は科学的根拠に基づき、多くのランナーに支持されている。本アプリは VDOT 式をウェブアプリとして誰でも簡単に利用できるようにする。

### 2.2 目標
| 目標 | 指標 |
|-----|------|
| アクセシビリティ | PWA 対応によるオフライン利用 |
| ユーザビリティ | 3クリック以内で結果表示 |
| 精度 | Daniels-Gilbert 式に基づく計算 |

---

## 3. 機能要件

### 3.1 コア機能

#### FR-001: ペース入力
- **説明**: ユーザーがキロあたりのペース（分・秒）を入力できる
- **入力方法**: ドロップダウン選択
- **範囲**: 分 1〜10、秒 0〜59
- **優先度**: 必須

#### FR-002: 距離入力
- **説明**: ユーザーが走行距離を入力できる
- **入力方法**: 数値入力 + プリセットボタン（5K, 10K, ハーフ）
- **範囲**: 1〜50km（小数第4位まで）
- **優先度**: 必須

#### FR-003: VDOT 計算
- **説明**: 入力されたペースと距離から VDOT 値を計算する
- **アルゴリズム**: Daniels-Gilbert 式
- **出力**: VDOT 値（小数第1位）
- **優先度**: 必須

#### FR-004: レースタイム予測
- **説明**: VDOT 値から各距離のレースタイムを予測する
- **対象距離**: 5K, 10K, ハーフマラソン, フルマラソン
- **出力形式**: 時:分:秒
- **優先度**: 必須

### 3.2 拡張機能

#### FR-005: PWA 対応
- **説明**: オフラインでも動作し、ホーム画面に追加可能
- **要素**: manifest.json, Service Worker
- **優先度**: 任意

## 4. 非機能要件

### 4.1 パフォーマンス
| 項目 | 要件 |
|-----|------|
| 初回読み込み | < 3秒（3G回線） |
| 計算処理 | < 100ms |
| Lighthouse スコア | パフォーマンス > 90 |

### 4.2 対応環境
| 項目 | 要件 |
|-----|------|
| ブラウザ | Chrome, Safari, Firefox, Edge（最新2バージョン） |
| デバイス | デスクトップ、タブレット、スマートフォン |
| 画面幅 | 320px 以上 |

### 4.3 アクセシビリティ
- キーボード操作対応
- スクリーンリーダー対応（セマンティック HTML）
- 適切なカラーコントラスト

---

## 5. 制約事項

- フレームワーク不使用（Pure HTML/CSS/JS）
- サーバーサイド処理なし（完全クライアントサイド）
- 外部 API 依存なし

---

## 6. 用語集

| 用語 | 説明 |
|-----|------|
| VDOT | Daniels' V.O2 max の略。ランニング効率と VO2max を組み合わせた指標 |
| Daniels-Gilbert 式 | Jack Daniels と Jimmy Gilbert が開発したレース予測式 |
| PWA | Progressive Web App。ネイティブアプリのような体験を提供するウェブ技術 |
